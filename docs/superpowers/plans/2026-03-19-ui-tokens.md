# UI Tokens Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up a Vite + React project with shadcn/ui v4, Tailwind v4, and a complete `globals.css` token system matching the UI tokens design spec.

**Architecture:** Single `globals.css` file defines all design tokens as CSS custom properties in `:root`/`.dark` blocks, registered with Tailwind v4 via `@theme inline`. shadcn/ui components inherit the token system automatically. A token preview page verifies all tokens render correctly in both modes.

**Tech Stack:** Vite, React 19, TypeScript, Tailwind CSS v4, shadcn/ui v4, DM Sans (Google Fonts), JetBrains Mono (Google Fonts)

**Spec:** `docs/superpowers/specs/2026-03-19-ui-tokens-design.md`

---

```

```

- [ ] **Step 1: Write the complete globals.css**

Replace the contents of `src/styles/globals.css` with the full token file. Structure:

1. Imports (`@import "tailwindcss"`, `@import "tw-animate-css"`, `@import "shadcn/tailwind.css"`)
2. `@custom-variant dark (&:is(.dark *))`
3. `@theme inline` block — register all shadcn + custom colors, radius, fonts, spacing
4. `:root` block — light mode values (Warm Cream palette + all mode-independent tokens)
5. `.dark` block — dark mode color overrides + shadow override
6. `@media (prefers-reduced-motion: reduce)` — zero out spatial durations
7. `@layer base` — default body styles

